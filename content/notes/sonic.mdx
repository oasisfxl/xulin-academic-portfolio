SONIC遥操作
网络检查 → C++ deploy → PICO manager → 启动控制。
1. 检查网络
ip -br addr
确认有这两个网段：
192.168.2.xxx      # PICO/局域网
192.168.123.xxx    # G1 真机网段
如果 G1 那个 USB 网卡没起来，比如还是 DOWN，重新配置：
sudo ip link set enx6c1ff7df9277 up
sudo ip addr add 192.168.123.200/24 dev enx6c1ff7df9277
然后测一下 G1：
ping 192.168.123.161
ping 192.168.123.164
2. 确认 PICO XRoboToolKit
PICO 里 PC Service IP 仍然填服务器局域网 IP，比如：
192.168.2.170
不是 192.168.123.200。
XRoboToolKit 里确认：
Status: WORKING
Tracking: Head + Controller
Data/Control: Send
Pico Motion Tracker: Full body
3. 启动真机 C++ deploy
终端 1：
cd /home/xulinfu/projects/gr00t_n17_g1_dex3/GR00T-WholeBodyControl/gear_sonic_deploy
source scripts/setup_env.sh
./deploy.sh --input-type zmq_manager real
确认它显示：
Environment: real
Network Interface: enx6c1ff7df9277
Input Type: zmq_manager
看到确认提示后输入：
Y
等到 Init done。
4. 启动 PICO manager
终端 2：
cd /home/xulinfu/projects/gr00t_n17_g1_dex3/GR00T-WholeBodyControl
export DISPLAY=:1
source .venv_teleop/bin/activate
python gear_sonic/scripts/pico_manager_thread_server.py --manager --vis_vr3pt --vis_smpl
确认不再卡在：
waiting for body data...
5. 启动控制
人站好校准姿势，PICO 手柄按：
A+B+X+Y
终端应该出现：
[Manager] StreamMode switch: OFF -> PLANNER
然后按：
A+X
进入 POSE 模式。
急停
C++ 终端随时按：
O
PICO 侧按：
A+B+X+Y
可以切回 OFF。
下次如果只是做 sim2sim，不要跑 real，改成：
./deploy.sh --input-type zmq_manager sim
并且另开 MuJoCo：
cd /home/xulinfu/projects/gr00t_n17_g1_dex3/GR00T-WholeBodyControl
export DISPLAY=:1
source .venv_sim/bin/activate
python gear_sonic/scripts/run_sim_loop.py
